**Date**: 2026-04-06  
**Feature**: 034-flow-query

---

## 1. Input Types

### FlowStep

An anchor event in a flow query with per-step configuration.

| Field | Type | Default | Constraint |
|-------|------|---------|------------|
| `event` | `str` | (required) | Non-empty, no control characters |
| `forward` | `int \| None` | `None` | 0-5 when set; inherits top-level default if None |
| `reverse` | `int \| None` | `None` | 0-5 when set; inherits top-level default if None |
| `label` | `str \| None` | `None` | Display label; defaults to event name |
| `filters` | `list[Filter] \| None` | `None` | Per-step filters converted to segfilter format |
| `filters_combinator` | `Literal["all", "any"]` | `"all"` | How per-step filters combine |

**Invariants**:
- Frozen dataclass (immutable after construction)
- At least one of `forward` or `reverse` must be > 0 after normalization (FL5)
- `event` must be non-empty after stripping whitespace

**Normalization**: When a plain string is provided as input to `query_flow()`, it is normalized to `FlowStep(event=string)` with `forward`/`reverse` inherited from top-level defaults.

### Segfilter (Internal)

The legacy filter format used by flows per-step `property_filter_params_list`. Not a public type â€” produced by the internal converter.

| Field | Type | Notes |
|-------|------|-------|
| `property` | `dict` | `{"name": str, "source": str, "type": str}` |
| `type` | `str` | Property type: `"string"`, `"number"`, `"boolean"`, `"datetime"`, `"list"` |
| `selected_property_type` | `str` | Same as `type` (no typecast support in v1) |
| `filter` | `dict` | `{"operator": str, "operand": str\|list}` |

**Source mapping**: `"events"` â†’ `"properties"`, `"people"` â†’ `"user"`

## 2. Output Types

### FlowQueryResult

Result of an ad-hoc flow query. Frozen dataclass inheriting from `ResultWithDataFrame`.

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `computed_at` | `str` | `""` | ISO timestamp from API |
| `steps` | `list[dict[str, Any]]` | `[]` | Raw sankey step data (nodes + edges) |
| `flows` | `list[dict[str, Any]]` | `[]` | Raw top-paths flow data |
| `breakdowns` | `list[dict[str, Any]]` | `[]` | Top-level breakdown data |
| `overall_conversion_rate` | `float` | `0.0` | End-to-end conversion |
| `params` | `dict[str, Any]` | `{}` | Generated bookmark params |
| `meta` | `dict[str, Any]` | `{}` | Response metadata |
| `mode` | `Literal["sankey", "paths"]` | `"sankey"` | Query mode that produced this result |

**Lazy-cached properties** (not stored, computed on first access):

| Property | Return Type | Description |
|----------|-------------|-------------|
| `nodes_df` | `pd.DataFrame` | Flattened nodes: `step, event, type, count, anchor_type, is_custom_event, conversion_rate_change` |
| `edges_df` | `pd.DataFrame` | Flattened edges: `source_step, source_event, target_step, target_event, count, target_type` |
| `graph` | `nx.DiGraph` | NetworkX directed graph; nodes keyed as `"{event}@{step}"` |
| `df` | `pd.DataFrame` | Mode-aware: `nodes_df` for sankey, `paths_df` for top-paths |

**Convenience methods**:

| Method | Return Type | Description |
|--------|-------------|-------------|
| `top_transitions(n=10)` | `list[tuple[str, str, int]]` | N highest-traffic (source, target, count) edges |
| `drop_off_summary()` | `dict[str, Any]` | Per-step drop-off counts and percentages |

## 3. Bookmark Structure (Flat â€” No Sections)

### Flow Bookmark Params

Generated by `_build_flow_params()`. Flat top-level structure, NOT wrapped in `sections`.

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `steps` | `list[dict]` | (required) | Array of step dicts |
| `date_range` | `dict` | (required) | Flat date range from `build_date_range()` |
| `chartType` | `str` | `"sankey"` | `"sankey"` or `"paths"` |
| `flows_merge_type` | `str` | `"graph"` | `"graph"` for sankey, `"list"` for top-paths |
| `count_type` | `str` | `"unique"` | `"unique"`, `"total"`, `"session"` |
| `cardinality_threshold` | `int` | `3` | 1-50 |
| `version` | `int` | `2` | Fixed |
| `conversion_window` | `dict` | `{"unit": "day", "value": 7}` | `{"unit": str, "value": int}` |
| `anchor_position` | `int` | `1` | 1-indexed position of first anchor |
| `collapse_repeated` | `bool` | `False` | Merge consecutive duplicate events |
| `show_custom_events` | `bool` | `True` | Include custom events |
| `hidden_events` | `list[str]` | `[]` | Events to hide |
| `exclusions` | `list` | `[]` | Reserved for future use |

### Step Dict Structure

Each entry in the `steps` array:

| Field | Type | Notes |
|-------|------|-------|
| `event` | `str` | Event name |
| `step_label` | `str` | Display label (defaults to event name) |
| `forward` | `int` | Steps to show after (0-5) |
| `reverse` | `int` | Steps to show before (0-5) |
| `bool_op` | `str` | Filter combinator: `"and"` or `"or"` |
| `property_filter_params_list` | `list[dict]` | Segfilter-format filter dicts |

## 4. API Request/Response

### Request Body (POST /arb_funnels)

```json
{
    "project_id": 12345,
    "query_type": "flows_sankey",
    "bookmark": { /* flat flow bookmark params */ }
}
```

### Response (Sankey)

```json
{
    "computed_at": "2025-01-15T10:00:00",
    "metadata": {"min_sampling_factor": 1.0},
    "breakdowns": [],
    "steps": [
        {
            "nodes": [
                {
                    "event": "Login",
                    "type": "ANCHOR",
                    "anchorType": "NORMAL",
                    "totalCount": "100",
                    "isComputed": false,
                    "isCustomEvent": false,
                    "segments": [],
                    "conversionRateChange": 0.0,
                    "breakdowns": [],
                    "edges": [
                        {
                            "event": "Search",
                            "type": "NORMAL",
                            "step": 1,
                            "totalCount": "80"
                        }
                    ]
                }
            ]
        }
    ]
}
```

### Response (Top-Paths)

```json
{
    "computed_at": "2025-01-15T10:00:00",
    "metadata": {"min_sampling_factor": 1.0},
    "flows": [
        {
            "flowSteps": [
                {"event": "Login", "type": "ANCHOR", "totalCount": "100"},
                {"event": "Search", "type": "NORMAL", "totalCount": "80"}
            ],
            "segments": []
        }
    ]
}
```

**Note**: `totalCount` is always a string in the JSON response. Parse with `int()`.

## 5. Validation Rules

### Layer 1: Argument Validation (`validate_flow_args`)

| Code | Field | Rule |
|------|-------|------|
| FL1 | `event` | At least one event/step required |
| FL2 | `steps[i].event` | Each event name must be non-empty |
| FL3 | `forward` | Must be 0-5 |
| FL4 | `reverse` | Must be 0-5 |
| FL5 | `forward + reverse` | Must be > 0 (at least one direction) |
| FL6 | `cardinality` | Must be 1-50 |
| FL7 | `conversion_window` | Must be positive |
| FL8 | `last` | Must be positive (delegated to shared V7) |

### Layer 2: Flat Bookmark Validation (`validate_flow_bookmark`)

| Code | Field | Rule |
|------|-------|------|
| FLB1 | `steps` | Must be non-empty list |
| FLB2 | `steps[i].event` | Must be non-empty string |
| FLB3 | `count_type` | Must be in VALID_FLOWS_COUNT_TYPES |
| FLB4 | `chartType` | Must be in VALID_FLOWS_CHART_TYPES |
| FLB5 | `date_range` | Must be present dict |
| FLB6 | `version` | Must be 2 |
